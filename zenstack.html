Lightweight Web Development on Android

Termux + Node.js + Zero Dependencies

android webdev termux node zero-deps

You don't need a laptop to build real web applications. With an Android phone, Termux, and Node.js, you can create, test, and deploy full-stack projects. This guide shows you how to set up a zero-dependency workflow that's fast, secure, and works on limited bandwidth.

What you'll build: This setup is used to maintain Zen Stack — 12 lightweight JavaScript modules for routing, servers, testing, and auth. Total size: 125KB. No npm packages required.

Why Zero Dependencies?

  • Fast installs: Clone and run. No npm install waiting.
  • Better security: Fewer packages means a smaller attack surface.
  • Portable: Runs on low-spec devices, old Android versions, and slow connections.
  • Easy to audit: Every line of code is yours. No surprises from third-party updates.

Step 1: Set Up Termux and Node.js

Install Termux

Download Termux from F-Droid. The Play Store version is outdated and no longer maintained.

Update packages

pkg update && pkg upgrade

Install required tools

pkg install git vim nodejs

This gives you Git for version control, Vim for editing, and Node.js to run JavaScript.

Step 2: Get the Code

Enable storage access

termux-setup-storage

Clone the project

cd storage/shared
git clone https://gitlab.com/brixtonmavu/zen-stack
cd zen-stack

Run ls -1 *.js to see the modules. Each file is a self-contained library: vanillarouter.js for routing, vanillaserver.js for HTTP, microtest.js for testing.

Step 3: Run a Web Server

Create a new file called server.js:

vim server.js

Add this code:

const VanillaServer = require('./vanillaserver.js');
const app = new VanillaServer();

app.get('/', (req, res) => {
  res.html('<h1>Hello</h1><p>Running on Node.js with zero dependencies.</p>');
});

app.get('/api/status', (req, res) => {
  res.json({ status: 'ok', dependencies: 0 });
});

app.listen(8000, () => {
  console.log('Server running at http://localhost:8000');
});

Start the server:

node server.js

Open http://127.0.0.1:8000 in Chrome or any browser on your phone.

Step 4: Try the Interactive Demos

The project includes a script to generate demo pages for every module.

chmod +x generate-demos.sh
./generate-demos.sh

This creates 12 *-demo.html files. Open vanillarouter-demo.html to see a live router example. Each demo includes a button to download the raw source code.

Step 5: Make Changes

Every module is a single .js file with no build step.

  1. Open any file: vim vanillarouter.js
  2. Edit the code and save with :wq
  3. Restart node server.js to see changes
Who is this for? Students learning web development, developers who prototype on mobile, and engineers building for users on slow connections.
